# Calendar

## When to use

- 用户需要查看日历、日程入口或后续扩展事件管理能力时
- 任务和时间安排、日程规划或提醒有关时

## Guidelines

1. 当前阶段以日历入口和视图承载为主，避免假设完整事件编排能力已经上线。
2. 如果需要复杂日程创建，先明确当前支持范围，再决定是否降级为说明或占位流程。
